# 競合UIUXモニタリングシステム - Kiro実装プロンプト

このファイルをKiroのSpecモードに貼り付けて、段階的に実装を進めてください。

## Phase 1: 管理画面の基盤構築

Next.js 14 App Router で管理画面を構築してください。

### タスク
1. Next.js プロジェクト初期化 (apps/web)
2. Supabase クライアント設定
3. Google OAuth 認証フロー実装（@lifull.com ドメイン制限）
4. 共通レイアウト（サイドバー + ヘッダー）
5. ダッシュボード画面（変更件数サマリー、直近の変更リスト）

## Phase 2: サイト・ページ管理

### タスク
1. サービス一覧・登録・編集・削除 CRUD
2. 監視ページ一覧・登録・編集・削除 CRUD
3. ページのPC/SP切り替え設定
4. ページ種別（一覧/詳細）設定

## Phase 3: スクレイピングエンジン

### タスク
1. Playwright スクリーンショット取得の完全実装
2. DOM構造抽出 + 物件固有情報除外ロジック
3. DOM差分検知（difflib ベース）
4. 画像差分生成（ピクセル比較 + 赤オーバーレイ）
5. ハイブリッド方式（404検知 + URL自動差し替え）
6. Supabase への結果保存

## Phase 4: AI分析パイプライン

### タスク
1. Gemini API によるカテゴリ分類
2. 変更サマリー生成（日本語3行）
3. 適用アドバイス生成
4. 結果のDB保存

## Phase 5: 通知・表示

### タスク
1. Slack 通知実装
2. 変更履歴一覧画面
3. Before/After 画像比較ビューア
4. DOM差分テキスト表示
5. AIアドバイス表示画面
6. カテゴリフィルタ

## Phase 6: 自動化・MCP

### タスク
1. GitHub Actions daily-scan ワークフロー完成
2. MCP Server 実装・テスト
3. 設定画面（Slack Webhook URL等）
4. 古いスクリーンショットの自動削除（30日）
