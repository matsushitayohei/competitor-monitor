# 競合UIUXモニタリングシステム - 要件定義

## FR-1: 対象サイト管理
- [ ] 監視対象のサービス（競合サイト）を登録・編集・削除できる
- [ ] 初期データ: SUUMO, athome, カナリー

## FR-2: 対象ページ管理
- [ ] 各サービスに対して監視対象ページ（URL）を登録・編集・削除できる
- [ ] ページごとにPC/SP切り替え設定ができる
- [ ] ページ種別（物件一覧/物件詳細）を指定できる
- [ ] 物件詳細URLが404の場合、一覧から自動で新URL取得（ハイブリッド方式）

## FR-3: スクレイピング・変更検知
- [ ] 毎日JST 6:00にGitHub Actionsでスキャンを実行する
- [ ] Playwrightでスクリーンショットを取得する（PC: 1280px / SP: 375px）
- [ ] DOM構造の差分を検知する
- [ ] 画像のピクセル差分を生成する
- [ ] 物件固有情報は除外ルールでフィルタする

## FR-4: 変更履歴・Before/After表示
- [ ] 検知した変更を時系列で一覧表示する
- [ ] Before/Afterスクリーンショットを左右並べて表示する
- [ ] 差分箇所をハイライト表示する
- [ ] DOM差分のテキスト表示（追加: 緑、削除: 赤）
- [ ] カテゴリでフィルタできる

## FR-5: AI分析・アドバイス
- [ ] 変更検知時にGemini APIで変更内容を分析する
- [ ] カテゴリ自動分類: CRO/広告商品/SEO/AI/その他
- [ ] LIFULL HOME'Sへの適用可否・優先度をアドバイス生成
- [ ] アドバイス内容: 概要、推定意図、適用提案、優先度、想定効果

## FR-6: Slack通知
- [ ] 変更検知時にSlack Webhookで通知する
- [ ] 通知内容: サービス名、カテゴリ、AIサマリー、詳細リンク
- [ ] Webhook URLは管理画面の設定から登録する

## FR-7: 認証
- [ ] Supabase Auth + Google OAuth
- [ ] @lifull.com ドメインのみ許可

## FR-8: MCP Server（Kiro連携）
- [ ] get_recent_changes: 直近N日間の変更一覧取得
- [ ] get_change_detail: 特定変更の詳細取得
- [ ] get_competitor_summary: サービス別変更傾向サマリー
- [ ] search_changes: カテゴリ・キーワード検索

## NFR-1: パフォーマンス
- [ ] 管理画面ページ表示: 3秒以内
- [ ] スクレイピング: 1サイト60秒以内

## NFR-2: コスト
- [ ] 月額$50以下（無料枠活用）

## NFR-3: データ保持
- [ ] Before/After画像: 最新30日分保持
